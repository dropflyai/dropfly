# Customer Success Brain -- Patterns

## Purpose

This directory contains reusable engagement patterns that encode proven solutions to recurring customer success challenges. Each pattern is a self-contained document describing the problem, context, solution, implementation guidance, metrics, and known trade-offs.

---

## Pattern Index

| Pattern | File | Problem It Solves |
|---------|------|-------------------|
| Onboarding Pattern | `onboarding_pattern.md` | How to structure customer onboarding to minimize time-to-value and maximize adoption |
| Churn Prevention Pattern | `churn_prevention_pattern.md` | How to detect at-risk customers early and intervene effectively before renewal |
| Expansion Pattern | `expansion_pattern.md` | How to identify and execute expansion opportunities to grow NRR |

---

## How to Use Patterns

1. **Identify the engagement challenge**: What customer lifecycle problem are you solving?
2. **Find the matching pattern**: Check this index for a pattern that addresses your challenge
3. **Adapt to your segment**: High-touch and tech-touch implementations differ; adapt accordingly
4. **Measure outcomes**: Every pattern includes success metrics. Track them to validate effectiveness
5. **Contribute back**: If you discover a new pattern or improve an existing one, update this directory

---

## Pattern Format

Each pattern follows this structure:

- **Problem**: The specific customer success challenge this pattern addresses
- **Context**: When this pattern applies (segment, lifecycle stage, engagement model)
- **Solution**: The pattern's approach with implementation details for each touch model
- **Metrics**: How to measure pattern effectiveness
- **Consequences**: Trade-offs, resource requirements, and known limitations
- **Related Patterns**: Other patterns that complement or conflict with this one
