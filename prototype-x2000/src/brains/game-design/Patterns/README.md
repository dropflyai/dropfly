# Game Design Brain -- Patterns

## Purpose

This directory contains reusable game design patterns that encode proven solutions to recurring design challenges. Each pattern is a self-contained document describing the problem, context, solution, implementation variations, metrics for success, and known trade-offs. Patterns are grounded in both academic game design theory and practical industry implementation.

## How to Use Patterns

1. **Identify the design challenge**: What problem are you solving in the game?
2. **Find the matching pattern**: Check this index for a pattern that addresses your challenge
3. **Adapt to genre and audience**: Patterns vary by game genre, platform, and target player type
4. **Playtest the implementation**: Patterns are starting points; playtest results drive refinement
5. **Contribute back**: If you discover a new pattern or validate a variation, update this directory

---

## Pattern Format

Each pattern follows this structure:

- **Problem**: The specific design challenge this pattern addresses
- **Context**: When this pattern applies (genre, platform, player type)
- **Solution**: The pattern's approach with implementation details and variations
- **Metrics**: How to measure pattern effectiveness
- **Consequences**: Trade-offs, risks, and known limitations
- **Related Patterns**: Other patterns that complement or conflict with this one

---

## Design Principles

All patterns in this directory adhere to the Game Design Brain's four pillars:

1. **Player-First Design**: Every pattern decision serves the player experience
2. **Systemic Thinking**: Patterns account for interconnected systems, not isolated features
3. **Evidence-Based Iteration**: Pattern implementations are validated through playtesting and analytics
4. **Ethical Engagement**: Retention through value delivery, never through exploitation
