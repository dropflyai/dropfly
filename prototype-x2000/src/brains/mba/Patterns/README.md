# MBA Brain -- Patterns Index

Repeatable strategic workflows for common business situations. Each pattern provides a structured approach that has been validated through application and can be adapted to specific contexts.

---

## What Are Patterns?

Patterns are documented, repeatable approaches to recurring business challenges. They encode institutional knowledge so that each new instance of a similar problem does not require starting from scratch.

### Pattern vs. Template
- **Pattern:** A strategic workflow (the thinking process and decision sequence)
- **Template:** A deliverable format (the output structure)

Patterns tell you HOW to think through a situation. Templates tell you WHAT to produce.

## Pattern Structure

Every pattern follows this standard structure:

1. **When to Use** -- Trigger conditions that indicate this pattern applies
2. **Prerequisites** -- What must be true before starting
3. **Phases** -- Sequential stages of the workflow
4. **Decision Gates** -- Go/no-go checkpoints between phases
5. **Key Metrics** -- What to measure at each phase
6. **Common Pitfalls** -- Mistakes to avoid
7. **Cross-Brain References** -- Which specialist brains to consult
8. **Exit Criteria** -- How to know the pattern is complete

---

## How to Use Patterns

1. **Identify the situation.** Read the "When to Use" section of each pattern to find the right one.
2. **Verify prerequisites.** Do not skip prerequisites -- they exist to prevent premature execution.
3. **Follow phases in order.** Each phase builds on the previous. Skipping phases creates gaps.
4. **Respect decision gates.** Gates are go/no-go checkpoints. If the gate criteria are not met, do not proceed.
5. **Adapt, do not blindly follow.** Patterns are starting points, not rigid scripts. Adapt to your specific context.
6. **Log application.** After using a pattern, log the experience in `Memory/FrameworkApplications.md` for future reference.

---

## Adding New Patterns

A new pattern should be created when:
- A workflow has been executed 3+ times with similar structure
- The workflow has enough complexity that documentation adds value
- The workflow crosses multiple phases with distinct decision points

To add a new pattern:
1. Follow the standard structure above
2. Include at least one real or realistic example
3. Add to this index
4. Log the addition in `Memory/FrameworkApplications.md`
