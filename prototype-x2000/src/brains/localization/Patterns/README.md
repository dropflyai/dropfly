# Localization Brain — Patterns Index

Reusable localization patterns for common scenarios. Each pattern
provides a step-by-step playbook with decision points, checklists,
and anti-patterns specific to the localization domain.

---

## Available Patterns

| Pattern | File | Use When |
|---------|------|----------|
| App Localization | `app_localization_pattern.md` | Localizing an application for the first time or adding a new locale |
| Market Expansion | `market_expansion_pattern.md` | Entering a new geographic market with localized product |
| L10n QA | `l10n_qa_pattern.md` | Setting up or improving localization quality assurance |

---

## How to Use Patterns

1. Identify the scenario that matches your current challenge
2. Read the full pattern before starting execution
3. Adapt the pattern to your specific context (team size, budget, timeline)
4. Follow the phases in order — each phase builds on the previous
5. Use the checklists as completion gates before moving to the next phase
6. Document deviations and outcomes in `Memory/ExperienceLog.md`

---

## When to Create a New Pattern

Create a new pattern when:
- A localization scenario repeats across 3+ projects
- The solution involves 5+ steps with specific ordering
- Getting the order wrong causes significant rework
- The pattern would benefit future team members or projects
