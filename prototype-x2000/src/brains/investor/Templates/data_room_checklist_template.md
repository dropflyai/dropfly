# Data Room Checklist — Template

## Purpose

This template provides a comprehensive checklist for organizing a virtual data room (VDR) for investor due diligence. A well-organized data room accelerates the fundraising process, signals operational maturity, and prevents delays caused by ad-hoc document requests. The data room should be prepared BEFORE investor meetings begin and updated continuously throughout the fundraising process.

## Data Room Management Best Practices

### Access Control
- **Read-only access** for all investors (no download permission initially)
- **Granular permissions** — different investors may see different sections
- **Activity tracking** — monitor who views what (most VDR platforms provide this)
- **Watermarking** — enable for sensitive financial documents
- **Revoke access** after fundraising concludes or if an investor passes

### Document Naming Convention
```
[Section#]_[Category]_[DocumentName]_[Date].[ext]

Examples:
02_Financial_MonthlyPnL_2024Q4.xlsx
03_Legal_CertificateOfIncorporation_2023-03-15.pdf
06_Customers_CohortAnalysis_2024-12.xlsx
```

### Update Protocol
- Update financial documents monthly during active fundraising
- Add new documents as investor requests come in
- Mark documents with date stamps to show currency
- Maintain version history for key documents (financial model, cap table)
- Designate a single person as data room manager to prevent chaos

### Common Data Room Mistakes

| Mistake | Prevention |
|---------|-----------|
| Room not organized before meetings start | Build data room 2-4 weeks before first investor meeting |
| Missing IP assignment agreements | Audit and collect before fundraising; this is a deal-killer |
| Financial data does not reconcile | Have Finance team verify all numbers against source of truth |
| Too much information (overwhelms investors) | Organize by priority; P3 items in supplementary only |
| Sensitive information without protection | NDA signed before data room access; watermark documents |
| Stale information | Date-stamp everything; update monthly |

## Usage Notes

- P1 items should be in the data room before the first investor meeting
- P2 items should be ready within 48 hours of request
- P3 items are nice-to-have; prepare if time permits
- Legal counsel should review all documents in the room before investor access
- Reference the `fundraising_round_pattern.md` for the complete fundraising workflow
- Archive the completed data room post-fundraising for future reference
