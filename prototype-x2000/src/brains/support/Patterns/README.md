# Support Brain — Patterns

## What Are Patterns?

Patterns are reusable solutions to recurring support challenges. Each pattern documents
a proven approach that can be adapted to specific contexts. Patterns encode institutional
knowledge so that teams do not reinvent solutions to problems that have already been
solved.

## Available Patterns

| Pattern | File | Use When |
|---------|------|----------|
| Support Scaling | `support_scaling_pattern.md` | Scaling support operations during rapid growth |
| Knowledge Base | `knowledge_base_pattern.md` | Building or overhauling a knowledge base |
| Incident Support | `incident_support_pattern.md` | Managing customer communication during incidents |

## Pattern Structure

Each pattern follows a consistent structure:

1. **Context** — When this pattern applies
2. **Problem** — What challenge it addresses
3. **Solution** — The proven approach
4. **Implementation** — Step-by-step guide
5. **Metrics** — How to measure success
6. **Anti-Patterns** — Common mistakes to avoid
7. **Examples** — Real-world applications

## How to Use Patterns

1. Identify the challenge you are facing
2. Find the matching pattern
3. Adapt the solution to your specific context
4. Implement with the provided steps
5. Measure using the defined metrics
6. Document any adaptations for future reference

## Contributing New Patterns

When a repeatable solution emerges from support work:
1. Document it using the pattern structure above
2. Add it to this directory
3. Update this README
4. Reference from Memory if it prevents a known loop

**Patterns are living documents. Update them as better solutions emerge.**
