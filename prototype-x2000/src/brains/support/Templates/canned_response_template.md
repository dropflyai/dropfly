# Template: Canned Response Library

## Purpose

This template provides the structure and examples for building a comprehensive library
of canned responses (macros, saved replies, templates) that agents use to respond
efficiently and consistently to common customer inquiries. Canned responses reduce
handle time, ensure quality consistency, and encode best practices into every agent's
workflow. Every response must be personalized before sending — canned responses are
starting points, not scripts.

---

## How to Use This Template

1. Copy the relevant section for each response type you need
2. Replace all `[bracketed text]` with your specifics
3. Customize tone to match your brand voice
4. Train agents on when and how to use each response
5. Review and update quarterly (or after every product release)
6. Track usage metrics: which macros are used most, least, and edited most

---

## Response Writing Standards

### Required Elements in Every Response

```
1. GREETING:       Personalized acknowledgment (use customer name)
2. ACKNOWLEDGMENT: Validate the customer's issue or question
3. BODY:           Solution, answer, or next steps
4. CLOSING:        Confirm resolution or set expectation for follow-up
5. SIGNATURE:      Agent name and support team identifier
```

### Tone Guidelines

| Do | Do Not |
|----|--------|
| Use the customer's name | Use "Dear Customer" or "Dear User" |
| Be warm but professional | Be overly casual or use slang |
| Use "I" for personal ownership | Use passive voice ("It has been noted") |
| Acknowledge frustration directly | Dismiss or minimize the issue |
| Offer specific next steps | Leave the customer guessing what happens next |

## Category 1: First Response Templates

### General First Response (Email)

```
Subject: Re: [Original subject]

Hi [Customer Name],

Thank you for reaching out to [Company] support. I'm [Agent Name], and
I'm happy to help with [brief restatement of issue].

[Solution or next steps]

[If more info needed:]
To help me investigate further, could you please share:
- [Specific info needed 1]
- [Specific info needed 2]
- [Screenshot if applicable]

[If resolved:]
This should resolve the issue. Please let me know if you have any
other questions — I'm here to help.

Best,
[Agent Name]
[Company] Support
```

### First Response — Bug Report

```
Hi [Customer Name],

Thank you for reporting this. I can see that [restate the issue in
your own words to confirm understanding].

I've reproduced this on my end and confirmed it's a bug. I've
filed this with our engineering team as a [priority level] issue.

[If workaround available:]
In the meantime, you can work around this by [workaround steps].

[If no workaround:]
I understand this impacts your workflow, and I'm sorry for the
inconvenience. I'll update you as soon as I have a timeline for
the fix.

Is there anything else I can help with in the meantime?

Best,
[Agent Name]
```

### First Response — Feature Request

```
Hi [Customer Name],

Thank you for sharing this idea — I appreciate you taking the time
to suggest it.

[If feature is planned:]
Great news: this is actually on our roadmap. While I can't share
a specific release date yet, I've added your feedback to the
feature request so our product team knows there's demand.

[If feature is not planned:]
I've passed this along to our product team with your use case
details. While I can't promise if or when this will be built,
customer feedback like yours directly influences our roadmap.

[If alternative exists:]
In the meantime, you might be able to accomplish something
similar by [alternative approach].

Thank you again for the suggestion!

Best,
[Agent Name]
```

## Category 2: Troubleshooting Templates

### Requesting More Information

```
Hi [Customer Name],

Thank you for reaching out. I'd like to help resolve this as
quickly as possible.

To investigate, I'll need a few details:

1. [Specific question — e.g., "What browser and version are you using?"]
2. [Specific question — e.g., "Can you share the exact error message?"]
3. [Specific question — e.g., "When did this start happening?"]
4. [Optional: "A screenshot of the error would be very helpful."]

Once I have this information, I'll be able to dig in right away.

Best,
[Agent Name]
```

### Common Troubleshooting Steps

```
Hi [Customer Name],

Thank you for the details. Let's try a few things to resolve this:

**Step 1:** [Clear, specific instruction]
**Step 2:** [Clear, specific instruction]
**Step 3:** [Clear, specific instruction]

After completing these steps, please let me know:
- Did the issue resolve? (If yes, great — we're all set!)
- Is the issue still occurring? (If so, please share [what to share])

I'm standing by to help further if needed.

Best,
[Agent Name]
```

### Escalation to Engineering

```
Hi [Customer Name],

Thank you for your patience as I investigated this.

After thorough troubleshooting, I've identified that this requires
attention from our engineering team. I've escalated your case with
all the details from our conversation — you won't need to repeat
anything.

Here's what to expect:
- Our engineering team will review this within [timeframe]
- I'll follow up with you as soon as I have an update
- If you have any additional information in the meantime,
  just reply here

I know waiting isn't ideal, and I appreciate your patience.

Best,
[Agent Name]
```

## Category 3: Account and Billing Templates

### Password Reset

```
Hi [Customer Name],

I can help you get back into your account.

To reset your password:
1. Go to [login page URL]
2. Click "Forgot Password"
3. Enter the email address associated with your account: [email]
4. Check your inbox for a reset link (also check spam/junk)
5. Click the link and create a new password

The reset link expires in [timeframe], so please use it promptly.

If you don't receive the email within 5 minutes, let me know and
I'll investigate further.

Best,
[Agent Name]
```

### Refund Request — Approved

```
Hi [Customer Name],

I've processed a refund of [amount] to your [payment method ending
in XXXX]. You should see this reflected in your account within
[5-7 business days].

[If applicable: Your account has been [adjusted/downgraded/canceled]
as requested.]

A confirmation email has been sent to [email].

Is there anything else I can help with?

Best,
[Agent Name]
```

### Refund Request — Denied

```
Hi [Customer Name],

Thank you for reaching out about a refund.

I reviewed your account and unfortunately, this request falls
outside our refund policy because [specific reason — e.g., "the
subscription was active for more than 30 days"].

Here's what I can offer instead:
- [Alternative 1 — e.g., "A prorated credit for the remaining period"]
- [Alternative 2 — e.g., "A free month extension to give you more time"]
- [Alternative 3 — e.g., "Downgrade to a lower plan"]

Would any of these work for you? I want to make sure we find a
fair solution.

Best,
[Agent Name]
```

## Category 4: Follow-Up Templates

### Follow-Up — Checking In

```
Hi [Customer Name],

I wanted to follow up on your recent support request regarding
[issue summary].

Were you able to resolve the issue with the steps I provided?

If everything is working well, no action is needed — I'll close
this ticket in [48 hours].

If you're still experiencing issues, just reply here and I'll
continue investigating.

Best,
[Agent Name]
```

### Follow-Up — Engineering Update

```
Hi [Customer Name],

I have an update on the issue you reported regarding [issue summary].

[If fix deployed:]
Our engineering team has deployed a fix for this issue. Could you
please try [action] and let me know if the problem is resolved?

[If in progress:]
Our engineering team is actively working on this. The current
expected timeline is [timeframe]. I'll continue to update you
as I learn more.

[If delayed:]
I want to be transparent — the fix is taking longer than expected
due to [reason]. The revised timeline is [timeframe]. I understand
this is frustrating, and I appreciate your patience.

Best,
[Agent Name]
```

### Closing — Issue Resolved

```
Hi [Customer Name],

Great to hear that's resolved!

As a quick summary: [1-sentence summary of issue and resolution].

If you ever run into this again, here's a helpful article:
[KB article link]

Thank you for being a [Company] customer. Don't hesitate to
reach out if you need anything in the future.

Best,
[Agent Name]
```

## Canned Response Management

### Naming Convention
```
[Category]_[Subcategory]_[Specific]
Example: troubleshooting_login_password-reset
Example: billing_refund_approved
Example: incident_active_first-response
```

### Review Cadence
- **Monthly:** Review most-used and most-edited macros
- **Quarterly:** Full library audit (accuracy, tone, relevance)
- **On product release:** Update all affected macros before release
- **On policy change:** Update all affected macros immediately

### Usage Metrics to Track
| Metric | Purpose |
|--------|---------|
| Usage count per macro | Which macros are most valuable |
| Edit rate per macro | High edit rate = macro needs improvement |
| CSAT for tickets using macro | Macro effectiveness |
| Unused macros | Candidates for removal or promotion |
