# Partnership Patterns — Index

## Purpose

This directory contains reusable patterns for common partnership scenarios. Each pattern is a structured playbook that captures the proven approach for a specific partnership challenge, based on repeated execution across multiple partnerships.

## When to Use Patterns

Patterns are used when you encounter a partnership scenario that has been solved before. Rather than designing from scratch, start with the appropriate pattern and adapt it to the specific context.

## Available Patterns

| Pattern | File | Use When |
|---------|------|----------|
| Partner Launch | `partner_launch_pattern.md` | Launching a new partnership from signed agreement to first revenue |
| Channel Scaling | `channel_scaling_pattern.md` | Scaling a channel partner program from initial cohort to mature operation |
| Ecosystem Building | `ecosystem_building_pattern.md` | Building a multi-partner ecosystem with network effects |

## Pattern Structure

Each pattern follows a consistent structure:

1. **Context**: When this pattern applies and what problem it solves
2. **The Pattern**: Step-by-step execution framework
3. **Key Milestones**: Critical checkpoints that must be reached
4. **Common Variations**: How the pattern adapts to different contexts
5. **Anti-Patterns**: What not to do (common mistakes in this scenario)
6. **Success Criteria**: How to know the pattern has been successfully executed

## How to Apply a Pattern

1. Read the full pattern before beginning execution
2. Assess which variations apply to your specific context
3. Adapt timelines and milestones to your organizational constraints
4. Review anti-patterns to avoid known failure modes
5. Track progress against key milestones
6. Document deviations and outcomes in `Memory/` for future pattern refinement

## Pattern Maintenance

Patterns are living documents. After each application, document:
- What worked as described
- What required adaptation
- What was missing from the pattern
- Suggested updates for future users

Submit pattern updates through the standard brain update process.
