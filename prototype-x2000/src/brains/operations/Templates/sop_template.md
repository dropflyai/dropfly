## Section 12: SOP Review Checklist

Before approving this SOP, verify:
- [ ] Purpose clearly stated
- [ ] Scope boundaries defined
- [ ] All roles identified with RACI
- [ ] Prerequisites listed completely
- [ ] Steps are sequential and unambiguous
- [ ] Each step has clear instructions a new person could follow
- [ ] Decision points have clear criteria
- [ ] Quality checks at appropriate points
- [ ] Error handling covers known failure modes
- [ ] Performance metrics defined
- [ ] Process tested by someone other than the author
- [ ] Review date scheduled

---

**An SOP is only valuable if it is accurate, accessible, and followed. Review SOPs quarterly. Update them when the process changes. Train new team members using them. Measure compliance. An outdated SOP is worse than no SOP -- it creates false confidence.**
