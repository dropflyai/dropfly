# Pattern: Social Media Crisis Response

## Pattern ID

`CRISIS-RESPONSE-001`

## Classification

Emergency response pattern for social media crises at Level 3 and above. Provides minute-by-minute protocol for the critical first 24 hours and multi-day recovery procedures.

---

## When to Use This Pattern

Activate this pattern when:
- A negative post about your brand is going viral (10K+ engagement and growing)
- Multiple similar complaints emerge simultaneously (pattern suggests systemic issue)
- Media outlets are reporting on a brand issue
- An employee or executive is involved in a public controversy
- A brand safety incident occurs (offensive content published, data breach, product safety)
- Social listening alerts indicate sentiment crisis threshold exceeded

---

## Prerequisites

This pattern assumes the following are already in place (see 08_crisis/crisis_management.md):
- [ ] Crisis team identified with contact information
- [ ] Escalation matrix documented and distributed
- [ ] Pre-approved response templates available
- [ ] Social listening alerts configured
- [ ] Legal review process for crisis communications established
- [ ] Content pause procedure documented

---

## Phase 1: Detection and Classification (T+0 to T+15 minutes)

### 1.1 Immediate Assessment

| Action | Owner | Time |
|--------|-------|------|
| Identify the crisis source (post, article, event) | First responder (social team) | 0-2 min |
| Screenshot and archive the source content | First responder | 2-3 min |
| Assess current reach and velocity | First responder | 3-5 min |
| Classify crisis level (1-5 per crisis taxonomy) | Social media manager | 5-10 min |
| Initiate escalation per matrix | Social media manager | 10-15 min |

### 1.2 Classification Checklist

Answer these questions to determine crisis level:

```
[ ] Is this a single complaint or a pattern?
    Single → Likely Level 1-2
    Pattern → Likely Level 3+

[ ] Is the content going viral (rapid engagement growth)?
    No → Likely Level 1-2
    Yes → Likely Level 3+

[ ] Are media outlets covering or inquiring?
    No → Likely Level 1-3
    Yes → Likely Level 4+

[ ] Is there a safety, legal, or regulatory dimension?
    No → Level per engagement volume
    Yes → Minimum Level 4

[ ] Is executive/board involvement required?
    No → Level 1-3
    Yes → Level 4-5
```

### 1.3 First 15 Minutes Action Summary

| If Level 3 | If Level 4 | If Level 5 |
|-----------|-----------|-----------|
| Alert social media manager | Alert social media manager + marketing VP + communications | Alert C-suite immediately |
| Begin monitoring escalation | Pause ALL scheduled content | Pause ALL scheduled content |
| Draft initial acknowledgment | Activate crisis communications team | Activate full crisis team |
| Continue normal operations (non-related) | Cancel non-essential meetings for crisis team | All hands on crisis |

## Phase 2: Initial Response (T+15 to T+60 minutes)

### 2.1 Content Pause

| Action | Platform | Method |
|--------|----------|--------|
| Pause scheduled posts | All platforms | Pause in scheduling tool |
| Review next 48 hours of scheduled content | All platforms | Check for tone-inappropriate content |
| Hold influencer posts (if applicable) | Contact influencer manager | Direct communication to creators |
| Pause paid campaigns | All ad platforms | Pause in ads manager |
| Disable auto-responses | Chatbots, auto-replies | Platform settings |

### 2.2 Internal Coordination

| Action | Owner | Time |
|--------|-------|------|
| Brief crisis team on facts known | Social media manager | T+15 min |
| Assign roles: monitor, draft, approve, execute | Crisis team lead | T+20 min |
| Begin fact-finding with relevant departments | Crisis team | T+20 min |
| Open dedicated crisis communication channel | Social media manager | T+15 min |
| Begin drafting initial public acknowledgment | Communications + social | T+20 min |

### 2.3 Initial Public Acknowledgment

**Draft and publish initial acknowledgment within 60 minutes of crisis detection.**

| Element | Content |
|---------|---------|
| Opening | Acknowledge awareness of the issue |
| Empathy | Express genuine concern for those affected |
| Action | State that you are actively investigating |
| Commitment | Promise to share updates as information becomes available |
| Channel | Post on platform where crisis is most active + owned channels |

**Template (Customize)**

```
We're aware of [brief, factual description of issue] and understand
the concern this has raised. We take this matter seriously and our
team is actively investigating.

We will provide an update as soon as we have more information.
If you've been directly affected, please reach out to
[support channel] so we can assist you personally.
```

**Approval Chain for Initial Acknowledgment**

| Crisis Level | Approval Required |
|-------------|-------------------|
| Level 3 | Social media manager + marketing VP |
| Level 4 | Marketing VP + communications + legal review |
| Level 5 | C-suite + legal + communications |

## Phase 3: Investigation and Substantive Response (T+1 to T+4 hours)

### 3.1 Fact-Finding Protocol

| Information Needed | Source | Priority |
|-------------------|--------|----------|
| What exactly happened? | Relevant department, product team | Critical |
| Who is affected and how many? | Customer data, support tickets | Critical |
| What caused the issue? | Technical team, operations | High |
| What is the current status? | Operations, product | Critical |
| What are we doing to fix it? | Technical team, leadership | Critical |
| Is there legal/regulatory exposure? | Legal counsel | High |
| What is the media angle? | Communications, PR | High |

### 3.2 Substantive Response Drafting

| Component | Content |
|-----------|---------|
| What happened | Clear, factual description (no minimizing, no speculation) |
| Who is affected | Scope of impact |
| What we are doing | Specific remediation actions |
| What affected individuals should do | Clear instructions for those impacted |
| How to get help | Specific support channels and contacts |
| When to expect updates | Committed update timeline |
| Accountability | Ownership of the issue (no blame-shifting) |

### 3.3 Substantive Response Approval and Publication

| Action | Owner | Timeline |
|--------|-------|----------|
| Draft substantive response | Communications + social team | T+1.5 hours |
| Legal review | Legal counsel | T+2 hours |
| Leadership approval | Appropriate authority per escalation matrix | T+2.5 hours |
| Format for each platform | Social media team | T+3 hours |
| Publish substantive response | Social media coordinator | T+3-4 hours |
| Prepare FAQ for community management | Social media team | T+3 hours |

### 3.4 Individual Response Protocol

| Comment Type | Response Approach |
|-------------|-------------------|
| Affected customer sharing experience | Empathetic response + direct support link |
| User asking questions | Direct to official statement or FAQ |
| Influencer/media amplifying | Monitor, prepare targeted outreach if needed |
| Constructive criticism | Acknowledge, reference actions being taken |
| Hostile/trolling | One professional response if high-visibility; otherwise ignore |
| Misinformation | Factual correction referencing official statement |

## Phase 4: Ongoing Management (T+4 to T+48 hours)

### 4.1 Monitoring Cadence

| Timeframe | Monitoring Frequency | Report To |
|-----------|---------------------|-----------|
| T+4 to T+12 hours | Every 30 minutes | Crisis team |
| T+12 to T+24 hours | Hourly | Crisis team lead |
| T+24 to T+48 hours | Every 2 hours | Marketing VP |
| T+48+ hours | Every 4 hours (if still active) | Marketing VP |

### 4.2 Update Communications

| Timing | Content | Channel |
|--------|---------|---------|
| T+6 hours | Progress update if new information | Same as initial response |
| T+12 hours | Status update (even if no new info: "still working on this") | Same as initial response |
| T+24 hours | Comprehensive update with actions taken | All owned channels |
| T+48 hours | Resolution update (if resolved) or progress update | All owned channels |
| Ongoing | Daily updates until fully resolved | All owned channels |

### 4.3 Stakeholder Management

| Stakeholder | Communication | Frequency |
|-------------|--------------|-----------|
| Crisis team | Dedicated channel updates | Continuous |
| All employees | Internal communication (email, Slack) | Daily minimum |
| Board/investors | Executive briefing | Per severity (Level 5: daily) |
| Media | Press statement via communications | Per inquiry |
| Partners/vendors | Direct communication if affected | As needed |
| Customers (direct) | Email/direct communication if affected | As resolution progresses |

---

## Phase 5: Resolution (T+48 hours to T+2 weeks)

### 5.1 Resolution Communication

| Element | Content |
|---------|---------|
| Summary | What happened and what was done |
| Resolution | How the issue was fixed |
| Prevention | What changes are being made to prevent recurrence |
| Remediation | What affected individuals will receive (refund, credit, etc.) |
| Accountability | Ownership and commitment going forward |
| Gratitude | Thank community for patience and feedback |

### 5.2 Content Resumption

| Phase | Timing | Content Type |
|-------|--------|-------------|
| Careful resumption | T+48-72 hours | Value-first content only (educational, helpful) |
| Gradual normalization | T+72 hours to T+1 week | Normal content mix, no promotional content |
| Full resumption | T+1 to T+2 weeks | Full content calendar, including light promotional |
| Re-activate paid | T+72 hours to T+1 week | Resume paid campaigns (awareness first, then conversion) |

---

## Phase 6: Post-Mortem and Prevention (T+2 to T+4 weeks)

### 6.1 Post-Mortem Document

| Section | Content |
|---------|---------|
| Incident summary | What happened, when, severity, scope |
| Timeline | Minute-by-minute chronology of detection through resolution |
| Response evaluation | What went well, what could improve, speed of response |
| Communication evaluation | Were communications timely, accurate, empathetic? |
| Decision review | Were escalation and approval decisions appropriate? |
| Outcome assessment | Final impact (sentiment change, follower impact, revenue impact) |
| Root cause analysis | What caused the crisis, could it have been prevented? |
| Process improvements | Specific changes to crisis protocols |
| Prevention measures | Actions to prevent recurrence |
| Pattern update | Updates to this crisis response pattern based on learnings |

### 6.2 Post-Mortem Meeting

| Element | Detail |
|---------|--------|
| Attendees | All crisis team members, relevant department heads |
| Duration | 60-90 minutes |
| Facilitation | Blameless retrospective format |
| Output | Documented learnings, assigned action items with owners and deadlines |
| Follow-up | 30-day check on action item completion |

## Pattern Anti-Patterns

| Anti-Pattern | Risk | Prevention |
|-------------|------|------------|
| Skipping initial acknowledgment | Silence perceived as guilt or indifference | Always acknowledge within 60 minutes |
| Waiting for perfect information | Delay appears as cover-up | Acknowledge uncertainty; update as info available |
| Single-channel response | Missing audience on other platforms | Respond on all active platforms |
| No content pause | Tone-deaf scheduled content during crisis | Automatic pause at Level 3+ |
| Skipping post-mortem | No learning, repeat same mistakes | Mandatory post-mortem within 2 weeks |
| Resuming promotional content too early | Perceived as callous, reignites anger | Minimum 72 hours value-only content before promotional |

*Speed, empathy, transparency, and accountability are the four pillars of effective crisis response. This pattern ensures you execute on all four consistently. Practice this pattern through simulations before you need it.*
