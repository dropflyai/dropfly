# Hackathon Pattern

## Pattern Summary

A complete execution playbook for running a developer hackathon -- virtual, in-person, or hybrid. Hackathons are the highest-engagement developer event format, producing working projects, community bonds, and product feedback in a compressed timeframe. This pattern covers the full lifecycle from 10 weeks before the event to 4 weeks after, including planning, promotion, execution, judging, and post-event follow-up.

---

## 1. Prerequisites

Before starting this pattern, confirm the following:

- [ ] Budget approved ($5K-50K depending on format and size)
- [ ] API/product is stable enough for developers to build with in 24-48 hours
- [ ] Starter templates and quickstart are tested and working
- [ ] Support team is available during the hackathon (including off-hours)
- [ ] Judging criteria and prizes are defined
- [ ] Legal review of contest rules (if offering prizes)

---

## 2. Team Requirements

| Role | Responsibility | Time Commitment |
|------|---------------|----------------|
| Event Lead | Overall coordination, budget, logistics | 40% for 10 weeks |
| Developer Advocate(s) | Content, workshops, mentoring | 60% for 6 weeks |
| Community Manager | Promotion, registration, engagement | 50% for 8 weeks |
| Engineers (2-3) | Mentoring, technical support, starter templates | 30% for 4 weeks |
| Designer | Branding, templates, swag design | 20% for 4 weeks |

---

## 3. Format Selection

| Format | Duration | Audience | Budget | Team Size | Best For |
|--------|----------|---------|--------|-----------|---------|
| **Virtual Sprint** | 24-48 hours | 100-1,000 | $5K-15K | 5-8 | Maximum reach, global participation |
| **Virtual Extended** | 5-7 days | 50-500 | $5K-20K | 5-8 | Working professionals, quality projects |
| **In-Person** | 24-48 hours | 30-150 | $15K-50K | 8-15 | Deep engagement, team building |
| **Hybrid** | 24-48 hours | 50-300 | $20K-50K | 10-15 | Best of both, complex logistics |

## 4. Timeline

### Phase 1: Planning (Weeks 1-4)

**Week 1: Strategy**
- [ ] Define hackathon goals (adoption, feedback, content, community)
- [ ] Select format (virtual/in-person/hybrid)
- [ ] Set dates (avoid major holidays and competing events)
- [ ] Define theme or challenge tracks
- [ ] Set budget and get approval
- [ ] Select submission platform (Devpost, custom)

**Week 2: Design**
- [ ] Define judging criteria and weights
- [ ] Design prize structure (grand prize, category prizes, participation)
- [ ] Define rules (team size, technology requirements, submission format)
- [ ] Design hackathon branding (logo, colors, templates)
- [ ] Create registration page
- [ ] Create Discord/Slack channels for the event

**Week 3: Content Preparation**
- [ ] Build starter templates (1 per supported language/framework)
- [ ] Test all starter templates from scratch (fresh machine test)
- [ ] Write hackathon-specific documentation (quick reference card)
- [ ] Prepare "Getting Started" workshop content
- [ ] Create demo project (shows what's possible)
- [ ] Prepare mentor briefing document

**Week 4: Logistics**
- [ ] Recruit judges (3-5 people: mix of internal + external)
- [ ] Brief judges on criteria and process
- [ ] Recruit mentors (engineers available during the event)
- [ ] Set up support channels with staffing schedule
- [ ] Configure submission platform (categories, fields, deadlines)
- [ ] If in-person: book venue, arrange catering, plan A/V

### Phase 2: Promotion (Weeks 4-8)

**Week 4-5: Soft Launch**
- [ ] Open registration
- [ ] Announce in community channels
- [ ] Email existing developer list
- [ ] Post on social media (first wave)

**Week 6-7: Amplification**
- [ ] Partner cross-promotion (sponsors, tech partners)
- [ ] Post on hackathon directories (Devpost, MLH, hackathon.com)
- [ ] Community member sharing campaign
- [ ] Social media content (2-3 posts/week)
- [ ] Targeted outreach to developer communities and universities

**Week 8: Final Push**
- [ ] "Last chance to register" campaign
- [ ] Early registration reminder emails
- [ ] Share inspiration projects and past winners
- [ ] Publish detailed schedule and workshop information

### Phase 3: Pre-Event (Weeks 9-10)

**Week 9: Final Preparation**
- [ ] Send participant prep email (environment setup, resources, schedule)
- [ ] Host pre-hackathon workshop (API overview, starter template walkthrough)
- [ ] Finalize mentor schedule and support rotation
- [ ] Test all infrastructure (submission platform, communication channels, APIs)
- [ ] Prepare contingency plans (API issues, platform outage, judge no-show)

**Week 10 (Event Week - Pre-Event):**
- [ ] Send final reminder with schedule and links
- [ ] Open event Discord/Slack channels
- [ ] Post welcome message with rules and resources
- [ ] Confirm all judges and mentors are ready

### Phase 4: Execution (Event Days)

**Event Day 1:**

```
KICKOFF (Hour 0):
  □ Welcome ceremony (15 min)
    - Welcome, introductions, thank sponsors
    - Explain theme, rules, and prizes
    - Share schedule and key deadlines
  □ Getting Started workshop (30 min)
    - Live coding with the API
    - Walk through starter templates
    - Show how to submit projects
  □ Team formation (15 min, if solo participants)
    - Pitch ideas, find teammates
    - Or use pre-formed teams

HACKING BEGINS (Hour 1):
  □ Mentors available in help channels
  □ Response time SLA: < 15 minutes for blocking issues
  □ Share tips and resources periodically in announcement channel

CHECK-IN #1 (Hour 6):
  □ "How's everyone doing?" in announcement channel
  □ Share common tips based on support questions
  □ Highlight interesting projects in progress

CHECK-IN #2 (Hour 12):
  □ Midpoint encouragement
  □ Remind about submission deadline and requirements
  □ Share code snippets for common patterns
```

**Event Day 2 (if 48-hour hackathon):**

```
MORNING CHECK-IN (Hour 18):
  □ Encourage teams to start preparing their demo
  □ Share submission template and requirements
  □ Remind about deadline (exact time and timezone)

FINAL HOURS (Hour 20-24):
  □ "2 hours remaining" announcement
  □ "1 hour remaining" announcement
  □ "Submissions are now closed" announcement

SUBMISSION CLOSE (Hour 24):
  □ Confirm all submissions received
  □ Late submission grace period (15 min, if needed)
  □ Announce judging timeline
```

### Phase 5: Judging (Hours 24-28)

```
JUDGING PREPARATION (30 min):
  □ Distribute submissions to judges
  □ Remind judges of criteria and scoring rubric
  □ Assign categories if judges have specialties

JUDGING (2-3 hours):
  □ Each judge reviews all submissions independently
  □ Score on each criterion (1-10 scale)
  □ Written feedback for each submission

DELIBERATION (30-60 min):
  □ Judges discuss top 5-10 projects
  □ Resolve ties and edge cases
  □ Select winners for each category
  □ Select grand prize winner
```

### Phase 6: Awards and Closing

```
AWARDS CEREMONY (60 min):
  □ Thank all participants
  □ Honorable mentions (3-5 projects)
  □ Category winners (with demo from each team)
  □ Grand prize winner (with longer demo)
  □ Announce prizes and distribution process
  □ Thank judges, mentors, sponsors
  □ Share next steps and community links
```

### Phase 7: Post-Event (Weeks 1-4 after)

**Week 1:**
- [ ] Send thank you emails to all participants
- [ ] Distribute prizes
- [ ] Send certificates of participation
- [ ] Collect feedback surveys
- [ ] Post winner announcements on social media

**Week 2:**
- [ ] Write hackathon recap blog post (with project highlights)
- [ ] Create video highlight reel (if recorded)
- [ ] Share project demos on social media
- [ ] Follow up with top builders (potential ambassadors)

**Week 3-4:**
- [ ] Analyze feedback surveys
- [ ] Compile metrics report
- [ ] Conduct internal retrospective
- [ ] Identify developers to nurture (potential champions)
- [ ] Update this pattern with lessons learned

## 5. Judging Criteria

| Criterion | Weight | Scoring Guide |
|-----------|--------|--------------|
| Technical Execution | 30% | Code quality, completeness, API usage depth |
| Innovation | 25% | Originality, creative problem-solving |
| Impact | 20% | Real-world usefulness, potential |
| Presentation | 15% | Demo quality, explanation clarity |
| API Integration | 10% | Depth and breadth of platform API usage |

**Scoring Scale:**
- 1-3: Below expectations
- 4-6: Meets expectations
- 7-8: Exceeds expectations
- 9-10: Exceptional

| Tier | Prize | Budget Allocation |
|------|-------|------------------|
| Grand Prize | $3,000-5,000 cash or credits | 30% of prize budget |
| 2nd Place | $1,500-2,500 | 15% |

## 9. Risk Mitigation

| Risk | Impact | Mitigation |
|------|--------|-----------|
| API outage during event | Critical | Dedicated on-call engineer, status page |
| Low registrations | High | Start promotion 6 weeks early, partner amplification |
| Low submission rate | Medium | Mid-event check-ins, encouragement, extended deadline |
| Judge no-show | Medium | Recruit 5 judges (need 3), backup judge identified |
| Cheating (pre-built projects) | Medium | Require Git history, demo must answer questions |
| Prize distribution issues | Low | Clearly define eligibility, international payment method |
