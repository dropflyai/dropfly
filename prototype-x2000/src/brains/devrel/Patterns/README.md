# DevRel Brain -- Patterns Index

## Overview

Patterns are reusable playbooks for recurring DevRel scenarios. Each pattern
codifies a proven approach with step-by-step execution guidance, decision
frameworks, and success criteria. Patterns are derived from the practices of
Stripe, Twilio, Google DevRel, Vercel, and other organizations with mature
developer relations programs.

---

## Available Patterns

### api_launch_pattern.md
**Use when:** Launching a new API, SDK, or major feature to the developer community.
**Covers:** Pre-launch preparation, documentation readiness, launch day execution,
post-launch follow-up, metrics and retrospective.

### community_growth_pattern.md
**Use when:** Growing a developer community from zero to self-sustaining or scaling
an existing community past a growth plateau.
**Covers:** Seeding strategy, platform selection, engagement programs, ambassador
pipelines, flywheel acceleration.

### documentation_pattern.md
**Use when:** Building comprehensive developer documentation from scratch or
restructuring existing documentation.
**Covers:** Diataxis framework implementation, information architecture, API
reference generation, tutorial design, documentation CI/CD.

---

## How to Use Patterns

1. Identify the scenario from the pattern index above
2. Read the full pattern file for the detailed playbook
3. Adapt timelines and team sizes to your specific context
4. Use the checklist sections as execution tracking tools
5. Reference the relevant module files for deeper theory

---

## Pattern Design Principles

- **Prescriptive:** Patterns tell you what to do, not just what to consider
- **Sequenced:** Steps are ordered by dependency and priority
- **Measurable:** Every pattern includes success criteria and metrics
- **Tested:** Patterns are derived from observed success, not speculation
- **Composable:** Patterns can be combined for complex initiatives
