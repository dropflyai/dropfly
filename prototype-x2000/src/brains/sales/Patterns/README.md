# Sales Brain — Patterns Directory

## What Are Patterns?

Patterns are reusable, battle-tested approaches to recurring sales scenarios. They
encode institutional knowledge — the collective wisdom of what works and what does
not in specific selling situations. Patterns are not rigid scripts; they are
strategic frameworks that can be adapted to context while maintaining their core
logic.

## Pattern Architecture

Each pattern follows a consistent structure:

1. **Context** — When does this pattern apply? What conditions trigger it?
2. **Challenge** — What makes this scenario difficult?
3. **Strategy** — What is the strategic approach?
4. **Execution** — Step-by-step tactical implementation
5. **Metrics** — How do you measure success?
6. **Anti-Patterns** — Common mistakes to avoid
7. **References** — Connections to other Sales Brain modules

## How to Use Patterns

1. Identify the scenario you are facing
2. Select the matching pattern
3. Review the full pattern before engaging
4. Adapt the execution steps to your specific context
5. Document deviations and outcomes for future pattern refinement
6. If a scenario does not have a pattern and recurs 3+ times, create one

## Pattern Quality Criteria

A pattern is considered valid when:
- It has been applied successfully in 3+ independent situations
- It addresses a scenario that recurs with meaningful frequency
- It provides specific, actionable guidance (not just principles)
- It has measurable success criteria
- It has been reviewed against Sales Brain methodology

## Adding New Patterns

When you identify a repeatable approach that works:
1. Document it using the pattern structure above
2. Include at least 2 real examples (anonymized)
3. Validate with at least one peer or manager
4. Submit for inclusion in this directory
5. Update this README with the new entry
