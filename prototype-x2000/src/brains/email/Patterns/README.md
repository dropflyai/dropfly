# Email Brain -- Patterns Index

Reusable email marketing patterns for common lifecycle scenarios. Each pattern provides a proven sequence of strategy, execution, and optimization steps with decision frameworks, timing guidance, and anti-patterns.

---

## Available Patterns

| Pattern | File | Use When |
|---------|------|----------|
| Welcome Sequence | `welcome_sequence_pattern.md` | Designing an onboarding/welcome email series for new subscribers or users |
| Re-engagement Campaign | `reengagement_pattern.md` | Building win-back flows, sunset policies, or list hygiene programs |
| Transactional Email Excellence | `transactional_email_pattern.md` | Designing receipts, notifications, password resets, and system emails |

---

## How to Use Patterns

1. Identify the lifecycle scenario that matches your current challenge
2. Read the full pattern before starting execution
3. Adapt timing, content, and segmentation to your specific audience and product
4. Follow the phases in order -- each phase builds on the previous
5. Use the checklists as completion gates before moving to the next phase
6. Test and iterate based on your own audience data (benchmarks are starting points, not targets)
7. Document deviations and outcomes in `Memory/README.md`

---

## When to Create a New Pattern

Create a new pattern when:
- An email program scenario repeats across 3+ projects or clients
- The solution involves 5+ emails with specific timing and logic
- Getting the sequence or timing wrong significantly impacts conversion
- The pattern would benefit future email program builds

---

## Pattern Quality Standards

- Every pattern must include timing recommendations based on industry research
- Patterns must reference compliance requirements (CAN-SPAM, GDPR)
- Patterns must include A/B testing recommendations
- Patterns must include measurable success criteria
- Patterns are reviewed quarterly and updated with new data

---

**Patterns are living documents. Update them as testing reveals better approaches.**
