# Video Production Pattern — End-to-End Workflow

## Context

This pattern governs the complete lifecycle of producing a video — from strategic brief through final delivery and distribution. It applies to any video type: educational, testimonial, product demo, brand content, or interview. The pattern is designed to prevent the three most common video production failures: unclear objectives leading to unfocused content, inadequate pre-production leading to expensive reshoots, and neglected distribution leading to zero ROI.

- [ ] Video brief completed (use `Templates/video_brief_template.md`)

- [ ] Brand guidelines reviewed for video-specific requirements
- [ ] Timeline agreed with all participants

## Phase 1: Pre-Production (40% of Total Effort)

### 1.1 Strategy and Brief

**Activities:**
- Complete video brief document
- Define success metrics (views, engagement, leads, conversions)
- Identify target audience and their viewing context (desktop vs. mobile, sound on vs. off)
- Determine platform requirements (aspect ratios, lengths, format constraints)
- Outline distribution and promotion plan

**Deliverable:** Approved video brief

### 1.2 Script Development

**Activities:**
- Research topic thoroughly (sources, data points, examples)
- Write outline (hook → main content → CTA structure)
- Draft full script with timing estimates
- Include visual direction notes (what viewers see while hearing narration)
- Review script for accuracy, brand voice, and engagement
- Revision cycle (maximum 2 rounds before approval escalation)

**Script Structure Template:**
```
HOOK (0:00-0:30)
[Visual] [Audio/Narration] [Text overlay]
- Pattern interrupt or curiosity gap
- Establish relevance ("If you're struggling with X...")
- Promise value ("By the end of this video, you'll know...")

MAIN CONTENT (0:30-[variable])
[Section 1]
[Visual] [Audio/Narration] [B-roll/graphics notes]
[Section 2] ...
[Pattern interrupt every 3-5 minutes for longer content]

CTA (last 30 seconds)
[Primary CTA] [Subscribe/visit/download]
[Secondary CTA] [Watch next video / join community]
```

**Deliverable:** Approved script with visual directions

### 1.3 Production Planning

**Activities:**
- Location scouting (or studio booking)
- Talent confirmation and preparation (talking points, wardrobe, availability)
- Equipment list (camera, lenses, lighting, audio, teleprompter)
- Shot list creation (every shot planned before shoot day)
- B-roll list (supplementary footage needed)
- Graphic/animation brief (lower thirds, title cards, charts, motion graphics)
- Call sheet creation (who, where, when, what to bring)

**Quality Gate 1:**
- [ ] Script approved by all stakeholders
- [ ] Shot list complete
- [ ] All logistics confirmed (location, talent, equipment)
- [ ] Call sheet distributed minimum 48 hours before shoot
- [ ] Graphic/animation brief submitted to designer

---

## Phase 2: Production (20% of Total Effort)

### Shoot Day Protocol

**Setup (1-2 hours before talent arrives):**
- Arrive early; set up lighting, audio, camera framing
- Test all equipment (record test footage, play back, check audio levels)
- Prepare teleprompter/cue cards if applicable
- Set up B-roll shots list for after main filming
- Brief talent on flow, timing, and key messages

**Main Filming:**
- Record safety take (full run-through for backup)
- Record section by section for easier editing
- Between sections: check audio levels, review footage, adjust as needed
- Capture multiple takes of critical moments (opens, CTAs, key statements)
- Allow talent to deliver both scripted and natural/improvised versions

**B-Roll Filming (After Main Filming):**
- Product shots, screen recordings, office/environment footage
- Hands doing things, close-ups, establishing shots
- More is better — B-roll is cheap to capture, expensive to reshoot

**Quality Gate 2:**
- [ ] All scripted sections captured (minimum 2 takes each)
- [ ] Audio quality verified on playback
- [ ] B-roll list fully captured
- [ ] Footage backed up to two locations before leaving set

---

## Phase 3: Post-Production (30% of Total Effort)

### 3.1 Edit

**Assembly Cut:**
- Select best takes for each section
- Assemble in sequence per script
- Add rough B-roll placement
- Rough timing check (within target duration?)

**Fine Cut:**
- Tighten pacing (remove pauses, filler, tangents)
- Finalize B-roll placements
- Add music bed (licensed, brand-appropriate)
- Add lower thirds, titles, graphics
- Add text overlays for key points
- Color correction and grading
- Audio mix (voice, music, SFX levels balanced)
- Add subtitles/captions (SRT file generation)

### 3.2 Review Cycle

**Review Protocol:**
- Round 1: Editor self-review against brief and checklist
- Round 2: Creative director / video lead review
- Round 3: Stakeholder review (content accuracy, brand compliance)
- Maximum 3 revision rounds (scope changes require new brief)

**Feedback Guidelines for Reviewers:**
- Provide timestamped feedback (e.g., "At 2:14, the graphic should show...")
- Distinguish between "must fix" and "nice to have"
- Reference the brief when requesting changes
- Use collaborative review tools (Frame.io, Vimeo Review)

### 3.3 Finalization

**Export Checklist:**
- [ ] Primary format for main platform (resolution, codec, bitrate)
- [ ] Alternate formats for secondary platforms (aspect ratios, lengths)
- [ ] Thumbnail(s) designed and exported (1280x720 minimum)
- [ ] Subtitles/captions file (SRT) exported separately
- [ ] Short-form clips identified and extracted
- [ ] Audio-only version if podcast distribution planned

**Quality Gate 3:**
- [ ] Final video reviewed against original brief
- [ ] Brand compliance checked (logo, colors, voice, tone)
- [ ] Accessibility: captions accurate, color contrast adequate
- [ ] Technical: audio levels normalized, no visual artifacts, clean transitions
- [ ] All platform versions exported and verified

---

## Phase 4: Distribution (10% of Effort, 50% of Value)

### Publishing Workflow

- Upload to primary platform with optimized metadata (title, description, tags, thumbnail)
- Schedule secondary platform uploads per distribution timing strategy
- Publish supporting content (blog embed, email, social teasers)
- Activate paid promotion if budgeted
- Share with employee advocacy network
- Notify stakeholders of publication

### Post-Publish Monitoring

- Monitor comments and respond within first 2 hours (engagement velocity signal)
- Track initial performance metrics at 1 hour, 24 hours, 48 hours, 7 days
- Compare to channel benchmarks
- Iterate on distribution if underperforming (boost, re-share, adjust metadata)

## Common Failure Modes

| Failure Mode | Prevention |
|-------------|-----------|
| Unclear objectives | Brief completion is mandatory before production |
| Scope creep during editing | Maximum 3 revision rounds; changes beyond scope require new brief |
| Poor audio quality | Always use external mic; test before filming; monitor during |
| Talking head fatigue | B-roll every 10-15 seconds; graphics; visual variety |
| No distribution plan | Distribution must be in the brief before production starts |
| Perfectionism paralysis | Set "good enough" criteria in advance; ship on schedule |
| Expensive reshoots | Thorough shot list and multiple takes prevent this |

## Post-Pattern Actions

1. Log production outcomes in `Memory/README.md`
2. Update cost tracking spreadsheet with actual costs
3. Schedule performance review at 7-day and 30-day marks
4. Identify repurposing opportunities and queue derivative production
5. Conduct brief post-mortem: what went well, what to improve
