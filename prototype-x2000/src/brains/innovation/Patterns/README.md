# Innovation Brain -- Patterns

## Overview

This directory contains reusable innovation workflow patterns. Each pattern encodes
a proven approach to a recurring innovation challenge. Patterns are not rigid templates --
they are adaptable frameworks that should be customized to context.

## Available Patterns

| Pattern | File | Use When |
|---------|------|----------|
| Lean Experiment | `lean_experiment_pattern.md` | Running a structured experiment to validate an assumption |
| Venture Building | `venture_building_pattern.md` | Creating a new business venture from concept to graduation |
| Innovation Sprint | `innovation_sprint_pattern.md` | Running a time-boxed innovation sprint (1-2 weeks) |

## How to Use Patterns

1. Identify the innovation challenge you face
2. Select the most relevant pattern
3. Adapt the pattern to your specific context
4. Execute the pattern, capturing learnings
5. If the pattern needs modification, update it for future use

## Pattern Format

Each pattern follows this structure:
- **Context**: When to use this pattern
- **Problem**: What challenge it addresses
- **Solution**: Step-by-step process
- **Artifacts**: What it produces
- **Anti-patterns**: What to avoid
- **Examples**: Real-world applications
