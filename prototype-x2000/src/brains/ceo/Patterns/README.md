# CEO Brain Patterns

## Overview

Patterns are proven, reusable approaches to common CEO-level challenges.
Each pattern describes a recurring situation, the proven approach to
handling it, the brains involved, and the expected outcomes.

## Available Patterns

| Pattern | File | When to Use |
|---------|------|-------------|
| Business Building | `business_building_pattern.md` | Building a new product or business from scratch |
| Crisis Response | `crisis_response_pattern.md` | Responding to any organizational crisis |
| Strategic Pivot | `strategic_pivot_pattern.md` | Changing company direction |
| Multi-Brain Project | `multi_brain_project_pattern.md` | Coordinating 3+ brains on a project |

## How to Use Patterns

1. Identify the situation you are facing
2. Select the most relevant pattern
3. Adapt the pattern to your specific context
4. Follow the steps, consulting referenced modules for depth
5. Log learnings to Memory/ after completion

## Pattern Structure

Each pattern follows this format:
- **Situation:** When this pattern applies
- **Approach:** Step-by-step process
- **Brains Involved:** Which specialist brains participate
- **Quality Gates:** How to verify quality at each stage
- **Common Pitfalls:** What goes wrong and how to avoid it
- **Example:** A worked example

## Adding New Patterns

When a new reusable approach is discovered through project execution:
1. Document the pattern following the structure above
2. Add to this README
3. Log the discovery in Memory/
