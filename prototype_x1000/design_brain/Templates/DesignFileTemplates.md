# Design File Templates â€” Authoritative

Start with structure, not a blank canvas.
Templates ensure consistency and speed.

---

## Purpose

Design file templates exist to:
- ensure consistent file organization
- speed up project kickoff
- maintain naming conventions
- prevent missing components/states
- enable smooth handoff to engineering

Every project starts with structure.

---

## Recommended Tools

### Primary (Choose One)
| Tool | Best For | Platform |
|------|----------|----------|
| **Figma** | Teams, collaboration, web-first | Web/Desktop |
| **Sketch** | Mac teams, established workflows | macOS |
| **Penpot** | Open source, self-hosted | Web |

### Supporting Tools
| Tool | Purpose |
|------|---------|
| FigJam / Miro | Wireframes, flows, workshops |
| Storybook | Component documentation |
| Zeplin / Avocode | Handoff (if not using Figma) |

## File Organization Structure

### Project File Naming
```
[ProjectName]_[Type]_[Version]

Examples:
- Acme_DesignSystem_v1.0
- Acme_WebApp_v2.3
- Acme_MobileApp_v1.1
- Acme_MarketingSite_v1.0
```

## Figma-Specific Setup

### Auto Layout Settings
```
Spacing:
- XS: 4px
- SM: 8px
- MD: 16px
- LG: 24px
- XL: 32px
- 2XL: 48px

Padding (use spacing tokens):
- Cards: 16px or 24px
- Buttons: 8px 16px (vertical horizontal)
- Inputs: 12px 16px
- Sections: 24px or 32px
```

### Component Property Setup
```
Button Component Properties:
- Variant: Primary | Secondary | Outline | Ghost | Danger
- Size: XS | SM | MD | LG
- State: Default | Hover | Focus | Active | Disabled | Loading
- Icon Left: Boolean
- Icon Right: Boolean
- Icon Only: Boolean
```

### Naming Conventions
```
Frames:
- Screens: [Section]/[Screen Name]/[State]
  Example: Auth/Login/Default

Components:
- [Category]/[Component]/[Variant]/[State]
  Example: Forms/Input/Text/Error

Layers:
- Use semantic names (not "Frame 1234")
- Group related layers
- Lock layers that shouldn't move
```
