# Content Inventory Framework — Authoritative

Before you design, know what you're designing for.
A content inventory prevents designing containers without contents.

---

## Purpose

Content inventory exists to:
- catalog all content the product needs
- identify content gaps
- plan content production
- inform design decisions
- prevent empty state surprises

Design without content understanding produces hollow shells.

---

## Content Types

### Core Content
The primary content users interact with.
```
- User-generated content
- Dynamic data
- System-generated content
```

### Supporting Content
Content that helps users understand and act.
```
- Labels and headings
- Help text and tooltips
- Error messages
- Success messages
- Empty states
- Onboarding content
```

### Marketing/Conversion Content
Content that sells or explains.
```
- Landing page copy
- Feature descriptions
- Testimonials
- Pricing information
- CTAs
```

### Legal/Compliance Content
Required content.
```
- Terms of service
- Privacy policy
- Cookie notices
- Compliance statements
```

## Content Requirements by Type

### Headlines & Titles
```
| Location | Max Length | Style | Example |
|----------|------------|-------|---------|
| Page title | 50 chars | Sentence case | |
| Section header | 30 chars | Sentence case | |
| Card title | 40 chars | Sentence case | |
```

### Body Copy
```
| Location | Length | Purpose | Example |
|----------|--------|---------|---------|
| Page intro | 1-2 sentences | Orient user | |
| Feature desc | 2-3 sentences | Explain value | |
| Help text | 1 sentence | Assist input | |
```

### Buttons & CTAs
```
| Context | Length | Format | Examples |
|---------|--------|--------|----------|
| Primary | 2-4 words | Verb + noun | Create workflow |
| Secondary | 2-4 words | Verb + noun | View all |
| Destructive | 2-3 words | Verb + noun | Delete workflow |
```

### Error Messages
```
| Error Type | Components | Example |
|------------|------------|---------|
| Field error | What + How to fix | "Email required. Enter a valid email address." |
| System error | What + Impact + Action | "Save failed. Your changes weren't saved. Try again." |
| Not found | What + Alternative | "Page not found. Return to dashboard." |
```

### Empty States
```
| Context | Components | Example |
|---------|------------|---------|
| First use | What + Why + CTA | "No workflows yet. Automate tasks. Create workflow." |
| Filtered | Result + Alternative | "No matches. Adjust filters or clear all." |
| Deleted | What happened + Alternative | "Workflow deleted. View all workflows." |
```

## Dynamic Content Analysis

### User-Generated Content
```
| Field | Type | Min | Max | Required | Validation | Fallback |
|-------|------|-----|-----|----------|------------|----------|
| Name | Text | 1 | 100 | Yes | Alphanumeric | - |
| Description | Text | 0 | 500 | No | None | "No description" |
| Avatar | Image | - | 5MB | No | jpg/png | Default avatar |
```

### System-Generated Content
```
| Field | Generated By | Format | Example |
|-------|--------------|--------|---------|
| Created date | System | Relative | "2 hours ago" |
| Status | System | Badge | "Running" |
| Run ID | System | UUID | "abc123" |
```

## Anti-Patterns (Avoid)

- Designing before knowing content needs
- Lorem ipsum in final designs
- Missing error/empty states
- Inconsistent terminology
- No content ownership
- Last-minute content scramble
- Ignoring dynamic content constraints
