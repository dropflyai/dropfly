# User Journey Map Framework â€” Authoritative

A journey map tells the story of the user experience.
Without it, you design screens without understanding the narrative.

---

## Purpose

User journey maps exist to:
- visualize the complete user experience
- identify pain points and opportunities
- align team on user perspective
- reveal emotional highs and lows
- inform design priorities

Screens are moments. Journeys are stories.

---

## Journey Map Types

### Current State Journey
How users experience things today.
- Based on research
- Reveals pain points
- Documents workarounds

### Future State Journey
How users should experience things.
- Based on design vision
- Defines target experience
- Guides implementation

### Day-in-the-Life Journey
Broader context of user's day.
- Beyond your product
- Shows when/how product fits
- Reveals adjacent needs

---

## Journey Map Components

### 1. Persona
Who is taking this journey?

### 2. Scenario
What are they trying to accomplish?

### 3. Stages
Major phases of the journey.

### 4. Actions
What users do at each stage.

### 5. Thoughts
What users think at each stage.

### 6. Emotions
What users feel at each stage.

### 7. Touchpoints
Where/how users interact.

### 8. Opportunities
Where design can improve experience.

## Journey Map Workshop

### Process (2-3 hours)
```
1. DEFINE (15 min)
   - Select persona
   - Define scenario
   - Set scope

2. MAP STAGES (20 min)
   - Identify major phases
   - Name each stage

3. POPULATE (60 min)
   - Actions per stage
   - Thoughts (quotes if possible)
   - Emotions (use research)
   - Touchpoints

4. ANALYZE (30 min)
   - Mark pain points
   - Mark moments of truth
   - Identify opportunities

5. PRIORITIZE (15 min)
   - Rank opportunities
   - Assign ownership
```

## Journey Validation

### Before Accepting
- [ ] Based on research (not assumptions)
- [ ] Includes actual user quotes
- [ ] Covers full journey scope
- [ ] Team has reviewed
- [ ] Opportunities prioritized

### Ongoing Validation
- Revisit after design changes
- Update with new research
- Track metrics per stage

---

## Anti-Patterns (Avoid)

- Journey based on assumptions
- Missing emotional dimension
- Only mapping the happy path
- Too granular (every click)
- Too abstract (no specifics)
- Made once, never updated
- No connection to design decisions
